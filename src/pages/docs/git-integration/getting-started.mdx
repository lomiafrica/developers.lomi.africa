# Getting Started

This guide will help you set up your development environment and make your first contribution to lomi.

## Prerequisites

- Node.js (v14 or higher)
- npm or yarn
- Git
- A GitHub account

## Development Setup

### 1. Fork and Clone
```bash
# Fork via GitHub UI first, then:
git clone https://github.com/YOUR_USERNAME/api.lomi.africa
cd api.lomi.africa

# Add upstream remote
git remote add upstream https://github.com/lomi/api.lomi.africa
```

### 2. Install Dependencies
```bash
# Install packages
npm install

# Set up environment
cp .env.example .env
```

### 3. Configure Environment
```bash
# .env
LOMI_API_KEY=test_key_xxx
LOMI_WEBHOOK_SECRET=whsec_xxx
NODE_ENV=development
```

## Running the Project

### 1. Development Server
```bash
# Start server
npm run dev

# Server runs on http://localhost:3000
```

### 2. Run Tests
```bash
# Run all tests
npm test

# Run specific tests
npm test -- --grep "payments"

# Run with coverage
npm run test:coverage
```

### 3. Build Documentation
```bash
# Start docs server
npm run docs:dev

# Build docs
npm run docs:build
```

## Making Changes

### 1. Create Branch
```bash
# Update main branch
git checkout main
git pull upstream main

# Create feature branch
git checkout -b feature/my-feature
```

### 2. Development Workflow
```bash
# Make changes
# Run tests
npm test

# Run linting
npm run lint

# Format code
npm run format
```

### 3. Commit Changes
```bash
# Stage changes
git add .

# Commit with conventional message
git commit -m "feat(payments): add Wave provider support"
```

## Submitting Changes

### 1. Push Changes
```bash
# Push to your fork
git push origin feature/my-feature
```

### 2. Create Pull Request
- Go to GitHub
- Create new pull request
- Fill out PR template
- Request review

### 3. Update PR
```bash
# Update with upstream changes
git fetch upstream
git rebase upstream/main

# Force push if needed
git push --force-with-lease
```
